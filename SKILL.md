# 竞赛作品主题奥卡姆蒸馏法

> 给定 N 个参赛项目 → 奥卡姆剃刀削去冗余 → 提炼 ≤5 个核心主题 → 输出分点总结 + 行动指南

---

## 核心原则

**不是翻译作品介绍，而是蒸馏本质共性。**

输入是一堆参赛项目的截图/文本，输出是：

```
主题1 ← 项目A、项目C 的共性
主题2 ← 项目B、项目D 的共性
...
```

---

## 三步蒸馏法

### 第一步：扫读归因（Occam Step 1）

对每个参赛作品只回答一个问题：

> **"这个项目解决的最根本问题是什么？"**

用 **≤8个字** 回答。这强制剃掉所有营销语言。

| 项目 | ≤8字本质 |
|------|----------|
| 项目A | 多Agent对抗协作 |
| 项目B | AI当同事 |
| 项目C | 多智能体交叉验证 |
| 项目D | 技能权限审计 |

### 第二步：聚类归并（Occam Step 2）

将第一步的"本质标签"分组。规则：

- **不要按技术分类**（Web/CLI/Mobile）——这是噪音
- **按"解决什么人的什么问题"分类**
- 每组 ≤4 个项目，超出则拆分为子主题
- 每组提炼一个**核心主题名**

输出格式：

```
核心主题: "让AI Agent可信、可控、可协作"
  ├─ 项目A: 对抗验证 → "修流程不修模型"
  ├─ 项目C: 交叉验证 → "不盲信单一来源"
  └─ 项目D: 权限审计 → "安装即治理"
```

### 第三步：行动映射（Occam Step 3）

对每个核心主题，回答三个问题：

1. **我们的系统缺这个吗？**（是/否/部分）
2. **缺的话怎么补？**（本周/本月/长期）
3. **不缺的话怎么强化？**（做对了什么）

---

## 输出模板

### 一、内容主题（剃刀精炼）

```markdown
| 作品 | 票数 | ≤8字本质 |
|------|------|-----------|
| ... | ... | ... |
```

**奥卡姆结论**：`N个项目 = X个共同方向 —— [一句话概括]`

### 二、分点总结

```markdown
① [关键词]
核心：[1-2句核心总结]

② [关键词]
核心：[1-2句核心总结]
```

**排版规则**：
- 序号用 ① ② ③ ④
- 每条先写关键词（≤6字），再写核心总结
- 条与条之间空一行

### 三、行动指南

```markdown
### 🔥 本周可做
1. [借鉴某项目] → [具体行动]
2. ...

### 📅 本月可做
1. ...
2. ...

### 🎯 长期方向
1. [护城河级改造]
```

## 参考案例：某竞赛 16 项作品分析

### 输入
16 张参赛作品截图，涵盖多 Agent 协作、数字员工、研究分析、安全审计等领域。

### 输出（蒸馏后仅3个核心主题）

| 主题 | 代表作品 | 共性方向 |
|------|---------|---------|
| 对抗验证体系 | re-forge (131票) | 多Agent互检，不通过不上线 |
| AI原生协作 | Helio (130票) | AI有身份/邮箱/主动性 |
| 可信数据流 | Investment Detective (71票) + SkillTrust (73票) | 交叉验证 + 权限审计 |

**结论**：16个项目 → 3个方向 → 1个终极问题：**"如何让AI从能干活进化到可信可控"**

## 技巧备忘

| 技巧 | 说明 |
|------|------|
| **≤8字法则** | 每个项目用≤8字概括本质，强制去噪 |
| **票数≠质量** | 高票反映需求共识，低票可能更前瞻 |
| **找对立面** | 如果两个项目看起来相反，它们可能指向同一问题的两端 |
| **归因到人** | 最终归因到"解决谁的什么问题"，不要归因到技术 |
| **空行呼吸** | 分点总结之间必须空一行，视觉清晰 |

---

## 适用场景

- 黑客马拉松 / 创业大赛作品评审
- 竞品分析（多产品横向对比）
- 技术趋势扫描（从散点中找主线）
- 团队内部灵感板（参赛作品 → 自身系统改造清单）
