# 测试案例集

## 案例1：某开发者大会 16 项作品分析

### 输入
16 张参赛作品截图，内容涵盖：
- AI Agent 编码团队（多角色分工）
- AI 数字员工（身份/邮箱/主动感知）
- 投资研报分析（多智能体编排）
- 技能安全审计（权限治理）
- 其他辅助工具

### 第一步：扫读归因

| 作品 | 票数 | ≤8字本质 |
|------|------|-----------|
| re-forge | 131 | 多Agent对抗协作 |
| Helio | 130 | AI当同事 |
| Investment Detective | 71 | 多Agent交叉验证 |
| SkillTrust | 73 | 技能权限审计 |

### 第二步：聚类归并

```
核心主题: "让AI Agent可信可控可协作"
  ├─ re-forge: 对抗验证 → 修流程不修模型
  ├─ Investment Detective: 交叉验证 → 不盲信单一来源
  ├─ SkillTrust: 权限审计 → 安装即治理
  └─ Helio: 身份协作 → AI是同事不是工具
```

### 第三步：行动映射

| 项目 | 我们的系统缺这个吗 | 怎么补 |
|------|-------------------|--------|
| re-forge | 部分（无对抗验证） | 本周：加"空头分析师"反对者Agent |
| Helio | 部分（被动触发） | 本月：Agent主动扫描表格变化 |
| Investment Detective | 是（无色彩编码） | 本周：IB色彩编码层到研报分析 |
| SkillTrust | 是（无skill审计） | 本月：建立技能权限检查清单 |

### 输出验证

| 检查项 | 结果 |
|--------|------|
| 每个项目有≤8字标签 | ✅ |
| 核心主题≤5个 | ✅ (1个) |
| 输出模板完整 | ✅ |
| 行动指南有本周/本月/长期 | ✅ |
| 最终一句话总结 | ✅ |

### 评分：95/100 (A)

## 案例2：模拟某 AI 创业大赛 8 项作品

### 输入（虚构）
8 个参赛项目：
- 智能客服机器人（NLP）
- 代码审查助手（代码扫描）
- 医疗影像诊断（CV）
- 农业无人机巡检（CV+IoT）
- 金融风控系统（ML）
- 教育个性化推荐（推荐系统）
- 工业缺陷检测（CV）
- 语音情感识别（语音）

### 第一步：扫读归因

| 项目 | ≤8字本质 |
|------|----------|
| 智能客服 | 降低人工成本 |
| 代码审查 | 提高代码质量 |
| 医疗影像 | 辅助诊断决策 |
| 农业无人机 | 自动化巡检 |
| 金融风控 | 自动化风险判断 |
| 教育推荐 | 个性化学习路径 |
| 工业检测 | 替代人工质检 |
| 语音情感 | 情绪状态识别 |

### 第二步：聚类归并

```
核心主题1: "用AI替代重复性人工判断"
  ├─ 金融风控
  ├─ 工业检测
  └─ 代码审查

核心主题2: "用AI增强专业决策质量"
  ├─ 医疗影像
  └─ 教育推荐

核心主题3: "用AI延伸人类感官/行动半径"
  ├─ 农业无人机
  └─ 语音情感
```

**结论**：8个项目 → 3个方向。所有项目都在回答一个问题：**"AI替代还是增强人类能力？"**

### 输出验证

| 检查项 | 结果 |
|--------|------|
| 聚类按"替代/增强"而非"NLP/CV" | ✅ |
| 核心主题≤5个 | ✅ (3个) |
| 一句话总结到位 | ✅ |

### 评分：92/100 (A)

## 验证汇总

| 案例 | 得分 | 关键发现 |
|------|------|----------|
| 16项真实作品 | 95/100 | 高票项目集中在"可信可控" |
| 8项虚构作品 | 92/100 | "替代vs增强"是终极判别维度 |
